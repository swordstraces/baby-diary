# 🚀 宝贝日记 - 从零开始上线指南

> 本指南面向**完全没有小程序开发经验**的用户，手把手带你从注册到上线。

---

## 第一步：注册微信小程序账号

1. 打开 [微信公众平台](https://mp.weixin.qq.com/)
2. 点击右上角「立即注册」
3. 选择「小程序」
4. 用你的邮箱注册（**不能是已绑定公众号的邮箱**）
5. 完成信息登记：
   - **主体类型**：选「个人」最简单（不需要营业执照）
   - **身份证**：需要管理员扫码验证
6. 注册成功后，进入后台 → 「开发」→「开发管理」→「开发设置」
7. 找到 **AppID**（类似 `wx1234abcd5678ef90`），复制保存

> 💡 个人主体小程序不能做微信支付，但我们的「宝贝日记」不需要支付功能，完全够用。

---

## 第二步：下载微信开发者工具

1. 打开 [开发者工具下载页](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 选择 **Mac** 版本（稳定版）
3. 下载 `.dmg` 文件，双击安装
4. 打开开发者工具，用你的**微信扫码登录**

---

## 第三步：配置项目

### 3.1 导入项目

1. 打开微信开发者工具
2. 点击「+」号 →「导入项目」
3. 填写：
   - **目录**：选择 `baby-tracker` 文件夹（就是本项目根目录）
   - **AppID**：粘贴第一步获取的 AppID
   - **开发模式**：小程序
   - **后端服务**：微信云开发
4. 点击「确定」

### 3.2 填入 AppID

打开项目中的 `project.config.json`，找到第 28 行：

```json
"appid": "your-appid-here",
```

替换成你的 AppID：

```json
"appid": "wx1234abcd5678ef90",
```

保存文件。

---

## 第四步：开通云开发

1. 在开发者工具顶部工具栏，找到 **「云开发」** 按钮（一朵云图标）
2. 点击「开通」
3. 输入**环境名称**（比如 `baby-tracker`）
4. 选择**基础版（免费）**
5. 点击确定，等待创建完成（约 30 秒）
6. 创建完成后，记住你的 **环境 ID**（类似 `baby-tracker-xxxxx`）

### 4.1 配置环境 ID

打开 `miniprogram/app.js`，找到第 8 行附近：

```javascript
wx.cloud.init({
  traceUser: true
})
```

改成：

```javascript
wx.cloud.init({
  env: '你的环境ID',   // 比如 'baby-tracker-xxxxx'
  traceUser: true
})
```

保存文件。

---

## 第五步：初始化数据库

### 5.1 部署 init 云函数

1. 在开发者工具左侧文件树，展开 `cloudfunctions` 目录
2. 你会看到一个云图标（☁️）或文件夹图标
3. 右键点击 `init` 文件夹
4. 选择 **「上传并部署：云端安装依赖」**
5. 等待部署完成（第一次可能要 1-2 分钟）

> ⚠️ 如果 cloudfunctions 目录没有云图标，说明云开发环境还没关联。点击工具栏「云开发」→ 确认环境已创建。

### 5.2 调用 init 初始化数据库

1. 右键点击已部署的 `init` 云函数
2. 选择 **「云端测试」**（或「开启云函数本地调试」）
3. 点击「运行测试」
4. 查看返回结果，应该显示 7 个集合全部 `created` 或 `already_exists`

### 5.3 验证数据库

1. 点击工具栏「云开发」
2. 进入「数据库」
3. 确认看到以下 7 个集合：

| 集合名 | 说明 |
|--------|------|
| families | 家庭组 |
| members | 家庭成员 |
| babies | 宝宝信息 |
| records | 活动记录 |
| daily_reports | 每日报告 |
| comments | 评论 |
| invites | 邀请码 |

### 5.4 设置数据库权限（重要！）

对每个集合设置权限，让家庭成员能读写数据：

1. 在云开发控制台 → 数据库
2. 点击每个集合 → 「权限设置」
3. 选择 **「所有用户可读，仅创建者可读写」**（适合大部分集合）
4. 或者选择 **「自定义安全规则」**，输入：

```json
{
  "read": true,
  "write": "doc._openid == auth.openid"
}
```

> 💡 `records`、`comments` 集合建议用「所有用户可读，仅创建者可写」，这样家庭成员能看到彼此的记录。

---

## 第六步：部署其他云函数

按同样方式部署剩余 3 个云函数：

1. 右键 `login` →「上传并部署：云端安装依赖」
2. 右键 `generateReport` →「上传并部署：云端安装依赖」
3. 右键 `checkAlerts` →「上传并部署：云端安装依赖」

---

## 第七步：模拟器调试

现在项目应该可以运行了！

1. 在开发者工具中，确认**模拟器**已打开（右侧面板）
2. 你应该看到「宝贝日记」首页加载出来
3. 如果看到报错，查看下方**控制台(Console)**面板的错误信息

### 常见调试流程

| 操作 | 预期效果 |
|------|---------|
| 点击「首页」Tab | 显示今日概览页（快捷按钮 + 最近动态） |
| 点击「喂奶」快捷按钮 | 跳转到喂奶记录表单 |
| 填写表单后点击「保存」 | 记录保存成功，返回首页 |
| 切换到「时间线」Tab | 显示今日记录列表 |
| 切换到「报告」Tab | 显示每日报告（初始为空） |
| 切换到「我的」Tab | 显示设置页面 |

> 💡 首次使用时，还没有家庭和宝宝信息，部分页面会显示空状态。需要先在「我的」页面中创建家庭和添加宝宝。

---

## 第八步：真机预览

1. 点击开发者工具顶部 **「预览」** 按钮
2. 选择「扫码预览」
3. 用手机微信扫一扫生成的二维码
4. 在手机上实际体验小程序

> ⚠️ 真机预览的体验最接近真实用户，建议所有功能都在真机上测试一遍。

---

## 第九步：上传发布

### 9.1 上传代码

1. 确认所有功能正常
2. 点击开发者工具顶部 **「上传」** 按钮
3. 填写版本号（如 `1.0.0`）和备注（如「首次发布」）
4. 点击「上传」

### 9.2 提交审核

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入「管理」→「版本管理」
3. 在「开发版本」中找到刚上传的版本
4. 点击「提交审核」
5. 填写审核信息：
   - **功能页面**：选择首页
   - **服务类目**：选择「工具」→「备忘录」或「生活服务」
   - **测试账号**：（一般不需要）
6. 提交等待审核（通常 1-7 天）

### 9.3 发布上线

审核通过后：
1. 回到「版本管理」
2. 在「审核版本」中点击「发布」
3. 选择「全量发布」
4. 🎉 你的小程序就正式上线了！

## 🔧 可选配置

### 配置定时触发器（自动生成报告）

1. 打开云开发控制台 → 云函数
2. 点击 `generateReport` → 「定时触发器」
3. 添加触发器：Cron 表达式 `0 0 22 * * * *`（每天 22:00）
4. 保存

### 配置定时触发器（异常检查）

1. 点击 `checkAlerts` → 「定时触发器」
2. 添加触发器：Cron 表达式 `0 0 */2 * * * *`（每 2 小时）
3. 保存

### 替换正式 TabBar 图标

当前的 TabBar 图标是占位用的简易圆形，建议替换为正式设计的图标：

1. 准备 81×81px 的 PNG 图标（每种需要普通态和选中态各一个）
2. 替换 `miniprogram/images/` 下对应的文件：
   - `tab-home.png` / `tab-home-active.png` — 首页
   - `tab-timeline.png` / `tab-timeline-active.png` — 时间线
   - `tab-report.png` / `tab-report-active.png` — 报告
   - `tab-mine.png` / `tab-mine-active.png` — 我的

## ❓ 常见问题

### Q: 编译报错「文件未找到」
检查 `project.config.json` 中的 `miniprogramRoot` 是否为 `"miniprogram/"`。

### Q: 云函数调用报错「环境未初始化」
确认 `app.js` 中的 `env` 参数填写了正确的环境 ID。

### Q: 数据库操作报错「权限不足」
检查云开发控制台 → 数据库 → 对应集合的权限设置。

### Q: 模拟器白屏
检查控制台错误信息，最常见的可能是云开发环境 ID 未配置。

### Q: TabBar 不显示
- 确认 `images/` 下有完整的 8 个图标文件
- TabBar 页面路径必须与 `app.json` 中的 `pages` 列表匹配
- TabBar 最少 2 个、最多 5 个
